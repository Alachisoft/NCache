Alachisoft NCache 4.3

Step 2: Register Compact Types with NCache

NOTE: This feature is not available in NCache Express and Professional Edition.

Configure the Cache


Create a Local Cache named "myCSCache" as follows:


  1. Run NCache Manager.
  2. Create a new project.
  3. Create a new cluster. Select following options from the New Cluster Wizard:
    • Cache Node: It may be the current computer and may be some other if you want.
    • Cache name: myCSCache
    • Operation Timeout: 60 sec's (default)
    • Statistics Replication Interval: 600 sec (default)
    • Cluster Port: 8714
    • Maximum Size: 250 MB (default)
    • Eviction Policy: Priority based
    • Default Priority: Normal
    • Eviction Percentage: 5 (default) (In case of session it is recommended that turn off eviction)
    • Clean interval (sec.): 15
  4. Start the cluster.
  5. Verify that cache is running. For this, determine cache behavior by adding and removing objects from Statistics Window.
  6. Clear the cluster to make it ready for your application.

Registering the Compact Serializable Types


Perform the following steps to register your types with the configured cache:


  1. Stop the Cache if it is running.
  2. Click on the 'Compact Serialization' tab which will open up the following screen.


  1. Click 'Add Compact Types' button. A 'Select Compact Classes' dialog will appear.
  2. Browse for the assembly file that contains the compact types by clicking the 'Browse' button as shown in the figure.
  3. Select the classes you want to register with the cache from the loaded classes list.
  4. Press 'Add Class' button to add the classes in the selected classes list.
  5. Click 'Ok'.


  6. Newly added types will start appearing in the list box under the 'Compact Serialization' tab having Compact Type and Platform information.
  7. 'Remove' button will delete the selected type from the registered types list.


  8. Apply the changes to the cluster.

Note:

  • For using Compact Serialization, you are supposed to define a default public constructor (parameter less) in a compact serializable type.
An object registered for Data Sharing can't be registered for Compact Serialization at a same time.

Specifying InProc or OutProc Access Mode


Please note that this cache has been configured to be used as OutProc. You can verify this by looking at the 'Isolation Level' property on the Main tab of the Cache details window. If you want to change Isolation Level, do the following:

  1. Stop the cluster.
  2. Go to Main tab.
  3. Change Isolation Level to 'Low'. This sets it for InProc. 'High' means OutProc.
  4. Apply the changes to the cluster. You will notice that you no longer have the 'myCSCache node->Start' menu enabled. This is because, the cache is now supposed to start as part of your application process.
