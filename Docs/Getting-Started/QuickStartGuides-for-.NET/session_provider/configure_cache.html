Step 1: Configure the Cache

To use Session State Store Provider in your application, follow the steps mentioned below. You can either choose to create a Local Cache that resides only on your computer or a Clustered Cache involving multiple computers. For now, we'll create a clustered cache.


  1. Launch NCache Manager from Start->Programs->NCache->NCache Manager.
  2. Create a new project.
  3. Create a new cluster. Select following options from the New Cluster Wizard:
    • Cluster Name: myPartitionedCache
    • Cluster Type: Partitioned
    • Operation Timeout: 60 sec's (default)
    • Statistics Replication Interval: 60 sec (default)
    • Member Nodes: At least one node
    • Cluster Port: Any available port
    • Maximum Size: 1024MB (default)
    • Eviction: Priority Based (default)
    • Default Priority: Normal
    • Eviction Percentage: 5 (default)(In case of sessions, we recommend to turn off eviction)
    • Clean Interval (sec's.): 15 (default)
  4. Start the cluster.
  5. Verify that cluster is running. For this, determine cache behavior by adding and removing objects from Statistics Window.
  6. Clear the cluster to make it ready for your application.

Specifying InProc or OutProc Access Mode


Please note that this cluster has been configured to be used as OutProc. You can verify this by looking at the 'Isolation Level' property on the Main tab of the Cluster details window. If you want to change it to InProc, do the following:

  1. Stop the cluster.
  2. Go to Main tab.
  3. Change Isolation Level to 'Low'. This sets it for InProc. 'High' means OutProc.
  4. Apply the changes to the cluster. You will notice that you no longer have the 'myCache node->Start' menu enabled. This is because the cluster is now supposed to start as part of your application process.
